ツCKヤ DPS Meter — Privacy Policy

Last updated: 2026-05-31

This app is built on a simple principle: your data stays on your machine. There are no user accounts, no ads, and no third-party analytics or tracking of any kind. This page describes the small set of cases where data does leave your device, and how you can opt out.

1. Combat Log Files & Character Names

ツCKヤ DPS Meter reads Throne & Liberty combat log files from your local disk to calculate damage per second and build performance. These files contain in-game character names (yours and other players' in your party/raid). The app reads these files locally; their contents are never uploaded or transmitted to any server unless you explicitly use the features described in Sections 3 and 4 below.

2. Windows Username in File Paths

Because Throne & Liberty stores combat logs under your Windows user profile (for example, C:\Users\YourName\...), the file paths the app works with embed your Windows username. This path is used only locally for file I/O. It is not stored, logged, or sent anywhere — except that the sanitized config snapshot in the optional feedback feature (Section 3) strips full paths before sending. See below for details.

3. Optional In-App Feedback & Bug Reports

The app includes an optional feedback and bug-report feature. This feature only activates when you explicitly choose to submit a report. Nothing is sent silently or automatically.

When you submit a report, the following is sent to the app's backend:

  • An in-memory event buffer — roughly the last 200 app events (UI actions, parse results, errors) recorded in RAM since launch. No file contents are included.
  • App version number.
  • Basic OS information (Windows version, architecture).
  • A sanitized configuration snapshot — settings like parser mode, selected columns, and similar preferences. File paths (including those containing your Windows username) are stripped from this snapshot before it is sent.

To opt out: Simply don't use the feedback feature. There is no opt-in or account required to use the rest of the app; the feature is only invoked when you press "Send Feedback" or equivalent in the UI.

4. Party DPS Scoreboard (Party Feature)

The Party DPS tab lets a group of players share a live scoreboard. When this feature is in use, the app transmits combat result data (per-boss DPS scores and character names for the current session) to an owned Cloudflare Worker that powers the shared scoreboard. This data is:

  • Scoped to the active party session only.
  • Not linked to any real-world identity — only the in-game character name you choose to display.
  • Stored on the worker only as long as needed to serve the scoreboard — capped to the most recent encounters and automatically deleted as new ones arrive. Never sold or shared.

To opt out: Don't join or create a party. All other app features (local parse, build testing, history) work entirely offline without touching this service.

5. What We Don't Collect

  • No user accounts or sign-in of any kind.
  • No advertising or third-party analytics (no Google Analytics, Sentry, Mixpanel, etc.).
  • No telemetry collected passively in the background.
  • No payment information — the app is free and open source.
